## 우선순위 중간 — 스키마/설정 실제 데이터로 교체

### 4. TC DB 실제 스키마 반영
`config/schema/tc_oracle.yaml` → `config/schema/tc_schema.yaml` 로 실제 TC DB 스키마 교체:
```sql
-- TC DB에서 추출
SELECT TABLE_NAME, COLUMN_NAME, DATA_TYPE, COLUMN_COMMENT
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = 'TCDB'
ORDER BY TABLE_NAME, ORDINAL_POSITION;
```
추출 결과를 yaml로 변환 후 저장. (AGENT_GUIDE.md 섹션 10-1 참고)

> `app/infra/config/loader.py`의 `load_schema()`가 `tc_oracle.yaml`을 참조하고 있음 → `tc_schema.yaml`로 파일명 변경 필요

### 5. whitelist.yaml 실제 컬럼으로 업데이트
TC_EQUIPMENT, TC_EQP_PARAM, TC_EQP_RELINK 컬럼이 실제 DB와 맞는지 확인 후 수정.
보안팀 리뷰 필요.

### 6. ValueStore 실제 값 로드
`app/api/deps.py`의 `init_dependencies()`에 추가:
```python
eqp_rows = await tc_pool.fetch_all("SELECT DISTINCT EQPID FROM TC_EQUIPMENT LIMIT 5000")
value_store.load_values({"EQPID": [r["EQPID"] for r in eqp_rows]})
```

## 우선순위 중간 — 프론트엔드 연동

### 7. Vue.js 대시보드 테이블 뷰어 구현
백엔드 API 완성됨. 프론트엔드에서 연동:

```javascript
// GET /api/v1/tables → 테이블 드롭다운
// GET /api/v1/tables/{name}?page=1&LINEID=L01 → 테이블 데이터
// GET /api/v1/tables/{name}/download?format=excel → 다운로드
```

구현 항목:
- [ ] 테이블 선택 드롭다운
- [ ] 필터 입력 폼 (filterable 컬럼 기반 동적 생성)
- [ ] 조회 버튼 (requires_where=true면 필터 없을 때 경고)
- [ ] 테이블 컴포넌트 + 페이지네이션
- [ ] CSV / Excel 다운로드 버튼
